## Overwrite ZITADEL Defaults

See a description of all possible configuration options with their defaults [in the source code](https://github.com/zitadel/zitadel/blob/v2/cmd/defaults.yaml).
You can configure these values by passing one or multiple configuration yaml files to the zitadel command.
Also, you can overwrite the default values by setting environment variables.
To do so, prefix the key with *ZITADEL\ _*, uppercase the propery and join sections by an underscore _.
For example, the environment variable *ZITADEL\_DATABASE\_USER\_USERMAME* sets the value for the following yaml config:

```yaml
Database:
  User:
    Username: zitadel
```
